# Lab 01 — ICMP Ping Analysis

## Objective

Capture and analyze ICMP echo request and echo reply traffic generated by a controlled ping test.

## Skills Demonstrated

- Selecting the correct capture interface
- Filtering ICMP traffic
- Comparing echo requests and replies
- Reviewing sequence numbers and timing

## Scenario

Generate ping traffic from the lab workstation to an authorized destination such as the local gateway or a public test address.

## Lab Environment

Complete the environment details in [`../../docs/lab-environment-and-tools.md`](../../docs/lab-environment-and-tools.md).

## Capture Procedure

1. Start Wireshark on the active interface.
2. Run a controlled ping test.
3. Allow several requests and replies to complete.
4. Stop the capture.
5. Apply the ICMP filter and inspect the exchange.

## Display Filters

```text
icmp
```

## Analysis Checklist

- [ ] Identify echo request packets
- [ ] Identify echo reply packets
- [ ] Compare identifier and sequence values
- [ ] Review round-trip timing
- [ ] Check for missing replies

## Expected Observations

Each successful ping usually produces an ICMP echo request followed by a matching echo reply. Missing replies may indicate packet loss, filtering, or an unreachable destination.
